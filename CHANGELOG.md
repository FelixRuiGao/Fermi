# Changelog

All notable changes to Fermi are recorded here. Releases that pre-date
this file are on the [GitHub Releases page](https://github.com/FelixRuiGao/Fermi/releases).

The release workflow extracts the section whose heading exactly matches
the pushed tag (e.g. `## v0.3.2-alpha.3`) and uses it as the GitHub
Release notes. A missing or empty section fails CI.

## Unreleased

- Changed: writing or editing the plan/todo file now appears in the conversation as a single `Update Todos` step instead of being dropped from the transcript entirely. Previously these operations were hidden (only the Todos panel reflected them), so when the file ever landed in the wrong place there was no on-screen sign anything had happened. The step suppresses the path, line-count suffix, and inline diff, since the Todos panel already shows the actual checkpoints.
- Changed: the welcome screen wordmark now renders in your theme's accent color (and follows a custom accent if you've set one), instead of a fixed blue-violet. It also shows at every terminal width — previously it was hidden on terminals narrower than 72 columns.

## v0.3.13

- Added: `/key` command to manage provider API keys without editing files by hand. Pick a provider endpoint (group providers like Kimi/GLM/Qwen/MiniMax drill down to each region, which has its own key) and replace, remove, or import a detected key. Covers API-key and managed providers plus custom endpoints; OAuth providers stay on `/codex` and `/copilot`. Changes apply to the running session immediately. Removing a key whose value also comes from your shell environment now warns that it will reappear on next launch.
- Changed: `fermi init` now configures the API key before listing models — after you pick a provider endpoint, it asks whether to keep the saved key or replace it, then shows that endpoint's models.
- Changed: `fermi init` now supports Esc (and Ctrl+C) to go back exactly one step at any prompt; backing out of the first step cancels setup. The back hint is shown in each picker's help line.
- Fixed: the agent sometimes wrote its plan/todo file to the project root instead of the session artifacts directory, so the TUI Todos panel stayed empty and checkpoints never reached you. The system prompt named the bare filename `plan.md` several times up front (with no path) but only stated the actual location once, far away in the tool docs — so the model defaulted to the working directory. The planning step no longer names the file at all (it just says to externalize multi-phase work as a live todo list), and the Plan File section now leads with an imperative to create it at the artifacts path, calling out that a `plan.md` written anywhere else is invisible to the TUI.
- Added: text you select with the mouse now auto-copies to the clipboard when you finish the drag (copy-on-select), with a brief "Copied" toast in the top-right corner; the highlight stays so you can re-copy. Toggle it with `/autocopy` (default on); Cmd/Ctrl+C still works too.

## v0.3.12

- Fixed: the conversation flickered with a full-screen redraw at the end of every turn. When the assistant's last message finished, its rendered markdown ran a finalize pass that re-snapped the trailing block's height; because the conversation is pinned to the bottom, that size change re-anchored the scroll and forced a full-viewport repaint every turn. Assistant markdown now stays in streaming render mode after a turn ends (so there's no finalize re-snap), and the streaming height reservation is scoped to only the actively-streaming message — completed messages settle at their true height, with no per-turn repaint.
- Added: skill and MCP availability changes now queue non-waking system notices for the next agent turn. Notices preserve the order of separate user operations, list enabled/disabled skills, and list concrete MCP tool names that became available or unavailable so the agent treats the change as normal runtime configuration instead of assuming its earlier reasoning was wrong.
- Fixed: MCP reload now removes stale `mcp__*` tool schemas and executors after servers are removed or changed, and refreshes same-name schemas from the current MCP manager, so the agent no longer sees tools the current MCP configuration cannot execute correctly.
- Fixed: enabling or disabling an MCP server (and toggling skills) no longer flashes a "Working" indicator in the input box — these are configuration changes, not agent turns. The actual MCP reload is now serialized against the conversation so it can't overlap a running turn.
- Fixed: skill/MCP availability notices now reliably land in the conversation before your next message (in arrival order), and are suppressed entirely before your first message — a config change made before you start chatting never becomes the agent's first context.
- Changed: the MCP "failed to connect" toast no longer disappears after 8 seconds. It stays until the server reconnects (auto-clears) or you dismiss it with Ctrl+L.
- Changed: turning auto-update on with `/autoupdate` now runs an update check immediately, instead of waiting until the next launch.
- Fixed: interrupting a turn now preserves completed thinking once the assistant has started writing normal text; thinking is discarded only if it was still in progress or if the assistant was cut off mid tool-call argument stream. Compact cancellations also clean up consistently now: interrupting automatic or manual `/compact` always records an interrupted turn cleanly and leaves a context note that compaction was canceled.- Fixed: MCP tools broke prompt caching on the first two turns. The `skill` tool was reordered relative to MCP tools between turns, invalidating the provider's cached tools prefix.

## v0.3.11

- Added: `/mcp` and `/hooks` are now interactive picker commands. `/mcp` shows each server with live connection status (✓ green / ✗ red), and offers Reconnect, Disable/Enable, View tools drill-down, and Reload config — the picker hot-updates as servers finish connecting. `/hooks` lists each hook with Disable/Enable toggle and Reload hooks to pick up changes from disk without restarting the session.
- Added: MCP servers connect eagerly at startup (fire-and-forget while you read the welcome screen) instead of blocking on the first message. Failed servers show a dismissible toast in the top-right corner. The first user message appears immediately in the TUI — MCP readiness is awaited after it's written to the log.
- Added: `reload` tool — the agent can call it after writing SKILL.md, AGENTS.md, or MCP config to refresh skills, rebuild the system prompt, and reconfigure MCP servers in one shot, without restarting the session.
- Added: `/usage` command — shows per-section context breakdown (system prompt, conversation, tools, etc.) with token counts and percentages of the context budget.
- Added: `/stat` command — shows all-time token statistics across sessions (total input/output tokens, cache hits, cost estimates).
- Added: Thinking entries in the conversation can be expanded/collapsed inline (click or Enter to toggle). Long thinking blocks start collapsed with a one-line preview.
- Fixed: child-process stderr (from tools, hooks, MCP servers) no longer corrupts the TUI. All stderr output is redirected to a log file (`~/.fermi/logs/stderr-{date}.log`), keeping the terminal clean.
- Fixed: the hand (pointer) mouse cursor over clickable elements — the model name, permission, the Agents/Shells/Todos chips, selectable rows — now works reliably. Two separate issues: (1) after `/resume`, hovering a clickable never changed the cursor. The terminal mouse-pointer escape (OSC 22) is only emitted inside a render tick, and a settled resumed conversation leaves the renderer idle; a bare hover changes no on-screen state, so it scheduled no frame and the escape never went out (it worked on the still-ticking empty/startup screen, hence the "works sometimes" feel). Setting the pointer now requests a render so the change reaches the terminal. (2) After the terminal lost and regained focus, the cursor could intermittently stay stuck on a stale hand until you moused over another clickable and off it. Ghostty records a pointer shape set while unfocused but doesn't repaint the cursor, then de-duplicates the now-identical shape on focus regain — so the corrective reset was a silent no-op. Fermi now never changes the pointer while the terminal is unfocused, so the recorded shape stays consistent with what's on screen; the first hover after refocus re-resolves to a real change and repaints cleanly, with no focus-time fixup needed. (3) Hovering empty screen background — edges, corners, the gaps between panels — showed a hand. The pointer was auto-detected from an element's own mouse-down handler, and the full-screen click-to-dismiss/focus background counts as one; an explicit cursor now wins over that auto-detection, and the background opts out to the default arrow.
- Fixed: opening a `write_file` detail tab while the agent is still streaming a long file (hundreds of lines) made scrolling crawl — and it freed up the instant the write finished. The detail view rendered every line of the file as a live row and re-syntax-highlighted the whole thing on every streamed chunk, which saturated the render thread and starved scrolling. The detail tab is now virtualized (only the rows in view are highlighted and built), and syntax highlighting memoizes per line so an unchanged file body isn't re-tokenized on each chunk. On a 400-line write the per-rebuild cost drops ~16x (`scripts/bench-file-modify-render.ts`); inline diff/write previews benefit from the same per-line highlight cache. Output is unchanged — locked byte-identical to the full render by `opentui-src/components/entry/file-modify-body.virtualize.test.ts`.
- Changed: the custom-provider wizard (`/model → Add custom provider...`) now accepts the full endpoint URL straight from a provider's docs. Pasting `https://openrouter.ai/api/v1/chat/completions` (or any `.../chat/completions` / `.../v1/messages` URL) used to be saved verbatim, after which the SDK appended its own path and every request 404'd; the wizard now strips the recognized suffix down to the base URL the SDK expects. When the suffix identifies the wire protocol it also skips the protocol question — `.../chat/completions` → OpenAI-compatible, `.../v1/messages` → Anthropic-compatible — and tells you what it detected. "Edit endpoint / API key" normalizes the same way.
- Added: four new models, selectable from `/model` — **MiniMax M3** (1M context, multimodal, thinking toggle; on MiniMax Global/China and OpenRouter), **Kimi K2.7 Code** (256K, multimodal, always-on thinking; on Moonshot Global/China/Coding), **GLM-5.2** (1M context; on the GLM Coding Plan endpoints — z.ai's standalone API for it lands next week), and **Qwen3.7 Plus** (1M context, multimodal vision + video input, thinking toggle; on Qwen China/Singapore/US and OpenRouter).
- Changed: upgraded the vendored OpenTUI core/react from 0.3.1 to 0.4.1, including the five prebuilt native targets. 0.4.x moves the Yoga layout engine inside the native library (the old npm `yoga-layout` dependency is gone), so the terminal renderer now ships one less moving part. All Fermi rendering customizations are preserved — cursor handling, sub-cell mouse/scrollbars (SGR-pixels), coalesced markdown spacing, the Agents│Todos split panel, and the `/resume` full repaint — and the behavior is intended to be identical to before. (Internal: see `Docs/opentui-0.4.1-upgrade-plan.md`.)
- Fixed: a few long-standing markdown spacing glitches. Prose immediately followed by a fenced code block, table, or block quote — with no blank line between them in the source — now keeps exactly one separator row, and tables again get their leading separator row. Vertical scrollbar thumbs can also shrink to a half-cell minimum now, matching the horizontal axis.
- Changed: multi-agent sessions release the memory of finished one-shot sub-agents (typically 50-200MB in agent-heavy sessions). A completed one-shot agent's full session — including its log and every tool result in it — used to stay resident until you closed the root session; it is now dropped the moment the agent settles, the Agents panel keeps showing its final state, and opening its transcript tab reads the log back from disk on demand. Persistent agents are untouched and can still be revived.
- Changed: on long sessions, the per-stream-chunk entry patches and per-tool-execution state updates that scanned the whole session log now use verified index lookups (200-300x faster on a 10k-entry log), turning an O(n²)-per-turn cost into O(n). Behavior is locked byte-identical to the old scans by randomized equivalence tests; `scripts/bench-log-hotpaths.ts` documents the measurements.
- Fixed: rewinding past a compaction or summarization boundary no longer leaves holes in the model's context. Compaction archives the pre-compact window's full content to disk; a rewind that removed the compact marker revived those entries with their content still stripped, and the API projection silently skipped them — the model just never saw that part of the conversation again. Rewind now restores the archived content from disk for every summary/compact boundary it truncates.
- Changed: long-session memory use drops. Content covered by a `summarize_context` summary is archived to disk the moment the summary lands instead of staying resident until the next compaction (5-30MB per long session); the TUI transcript poller idles at 1s instead of 250ms when nothing is running, which also stops idle RSS from creeping up; large inline diffs no longer rebuild their hunk structure on every frame.
- Fixed: agents no longer stop "silently" in the GUI / VS Code extension. Four invisible failure modes are closed: (1) turn errors (model 4xx, balance, context overflow) now write an error entry into the session log from the runtime itself — every UI shows them, instead of only the terminal UI which used to write that entry as a client-side courtesy; (2) sub-agent approval requests now reach out-of-process UIs — previously the approval event was derived by polling the root session's log, which a sub-agent's ask never touches, so the UI showed "working" forever while the agent waited for an answer that could never be given; (3) a crashed `fermi --server` process now saves the session, tells the client why it died (`server.crashed`), and the VS Code webview shows an actionable error screen instead of freezing; (4) auto-resume turns (sub-agent completion, shell exit) no longer swallow their errors — they emit the same events and log entries as user-initiated turns.
- Changed: `turn.ended` now reports status `waiting` when a turn parks on a pending approval, instead of claiming `completed`. The VS Code extension shows "Waiting for your approval…" in that state. Unknown statuses degrade to the old behavior on legacy clients.
- Fixed: two turn submissions arriving in the same instant (e.g. two GUI messages in one stdin chunk) could slip past the turn lock and run concurrent model loops, corrupting turn bookkeeping. The lock now claims synchronously and queues claimants strictly in order.
- Added: server-side conversation projection over RPC (`session.getProjectedLog` / `session.getProjectedChildLog`). Remote UIs receive the same canonical conversation structure the terminal UI renders, instead of re-implementing entry pairing/filtering over the raw log — the VS Code extension already consumes it; the raw-log methods remain for older clients. The `ready` event now carries `protocolVersion` and a capability list so clients can feature-detect.

## v0.3.10

- Fixed: self-update on Windows no longer goes through a detached PowerShell helper that waited for fermi to exit and then replaced files — a chain that silently dropped the update whenever any other fermi instance (a GUI tab's server process, the VS Code extension, a second terminal) kept the executable locked, and that relaunched fermi in a separate console window. Updates now install in-process on every platform: files locked by a running image are renamed aside (allowed on Windows, unlike delete/overwrite — the same idiom Claude Code's native installer and rustup use), the new file moves in with rollback on failure, and leftovers are cleaned up on later launches. A failed apply keeps the download and retries on the next launch instead of discarding it. After an update, fermi restarts in the same console.
- Changed: `read_file`'s per-line truncation is now recoverable in-tool. The default cap rose 2000 → 5000 characters (long URLs, wide table rows, and minified-adjacent lines fit untruncated), and a new `max_line_chars` parameter raises it per call — the truncation notice states the longest line's length and the exact parameter value to re-read it in full. The shell-pipeline hint now appears only for lines too long to fit the 80,000-character per-call budget at all. `grep`'s per-line cap stays at 2000 (it's for locating; follow up with `read_file`).
- Fixed: PDF text extraction now preserves the document's line structure (one text line per line, pages separated by blank lines). Previously the extracted view could collapse into a single enormous line, which the per-line cap then truncated to a fragment — forcing the agent into shell workarounds to read the document at all.
- Fixed: reading PDF/DOCX/XLSX files was broken in every release binary — the document converter (markitdown-ts) eagerly loaded jsdom, whose worker lookup the bundler froze into a build-machine absolute path, so any document read on a user machine failed with `Cannot find module ... xhr-sync-worker.js`. Local builds masked the bug because the baked path exists on the build machine. Document projection is now built on direct converters: PDF via unpdf (serverless pdf.js — no worker file, no DOM polyfills), DOCX via mammoth + turndown (same engines as before), and XLSX/PPTX via purpose-built OOXML readers. Nothing in the chain resolves files at runtime anymore.
- Added: PPTX projection now works — slide text, tables, and speaker notes per slide, in presentation order (rearranged decks keep their displayed sequence, not the part files' creation order). It had never worked: the previous converter routed PPTX through an optional dependency that was never installed.
- Fixed: XLSX date cells respect the workbook's 1904 date system (`workbookPr date1904`, common in Mac-origin files) instead of always assuming the 1900 epoch, which would have shifted those dates by four years.
- Changed: dropped markitdown-ts, jsdom, the Vercel AI SDK, and the npm `xlsx` package (frozen at 0.18.5 with two unfixed CVEs — CVE-2023-30533, CVE-2024-22363) from the dependency tree. The binary is ~6 MB smaller, and the pdfjs worker asset is no longer shipped alongside it.
- Changed: Bun runtime upgraded 1.3.9 → 1.3.14 — five releases of upstream fixes baked into the shipped binary, with a Windows-heavy payload (fs.openSync numeric flags, ConPTY groundwork, native win-arm64 toolchain). Version is now pinned in one place (`.bun-version`, read by CI); building from source requires Bun ≥ 1.3.10.
- Fixed: on macOS/Linux, background processes that ignore SIGTERM no longer survive `/clear`, rewind, or session shutdown as orphans — the batch kill path now escalates to a process-group SIGKILL after 1.5s, mirroring what single-shell kill already did (and what Windows' force-kill always did).
- Fixed: `FERMI_OPENTUI_USE_THREAD` was silently ignored on Linux — the renderer forced single-threaded mode over any explicit setting while the startup diag log recorded the value you asked for. Defaults are unchanged on all platforms; the override now actually takes effect.
- Fixed: paths under your home directory now display as `~/...` on Windows/macOS even when the drive letter or profile casing differs, and a sibling directory sharing the home prefix (e.g. `/Users/felixfoo`) is no longer mis-shortened.
- Added: Windows arm64 and Linux arm64 release builds. Snapdragon/ARM Windows machines get a native binary instead of x64 emulation, and Linux arm64 covers Docker on Apple Silicon, ARM servers, and Raspberry Pi. `install.sh` / `install.ps1` now detect the machine architecture (the PowerShell installer reads the real OS architecture, so it picks arm64 even from an emulated x64 shell), the VS Code extension downloads the matching asset, and self-update follows automatically. Release CI gained the two arm64 runners plus a `fermi --version` smoke test on every freshly built binary.

## v0.3.9

- Fixed: sending a message while an approval prompt (or agent question) was pending could permanently break the session — the approved tool silently never ran, and every later turn failed with an API projection error until the session was resumed from disk. The pending ask now keeps the session in a true waiting state: messages queue and are delivered when work resumes, and the approved tool always executes. GUI/VSCode (server mode) were the most exposed since they have no UI-level guard.
- Fixed: pressing stop/interrupt while an approval prompt was pending through the server API (GUI/VSCode) erased the question without an outcome and corrupted the conversation record. All stop entry points now resolve a pending ask as "deny and stop" — same clean wrap-up the TUI's Esc already did.
- Fixed: sending a message to an idle or archived persistent sub-agent crashed its turn immediately (the agent reported `failed` without doing anything). All parent→child deliveries now go through the standard delivery path.
- Fixed: a new message arriving right after an approval was granted (but before work resumed) was silently dropped when sent through `turn`; it is now queued and delivered, and both resume entry points behave identically.
- Changed: user input during context compaction is now rejected with a hint ("Compacting context — try again in a moment") instead of being silently folded into the compaction summary. Automatic messages (sub-agent results, shell exits) arriving mid-compact are held and delivered after the compaction marker.
- Fixed: `/compact` no longer leaves bookkeeping behind that could trigger a ghost auto-turn with no new input.
- Fixed: interrupting the agent right as it summarized context could leak the half-finished summary into a later, unrelated turn.

- Fixed: pressing Enter in a picker's custom-input field (e.g. /review's "Custom review instructions", /summarize_hint's level inputs) did nothing and cleared the field — broken since the feature shipped in v0.3.7. The deferred re-accept ran against the previous render's state and re-entered input mode instead of submitting; Enter now submits directly.
- Fixed: `/review` picker lost note text and custom-input text in every path — the user message bubble showed bare `/review` instead of the user's instructions. Root cause: commands that registered `options` had their picker intercepted before the handler ran; the accept path serialized the result to a string and dropped the note. Unified 7 commands (`/review`, `/summarize_hint`, `/permission`, `/rewind`, `/theme`, `/diff`, `/autoupdate`) to use handler-level `promptCommandPicker` exclusively, which returns structured `{ value, note }` through a resolver. `promptCommandPicker` now accepts an optional `{ title, allowNote }` config.
- Fixed: background shells (dev servers, watchers) are no longer leaked as orphan processes when the TUI exits — quitting, fatal errors, and SIGHUP/SIGTERM now kill all tracked shells before the process dies. Previously the cleanup rode on an un-awaited `session.close()` that never reached its kill step.
- Changed: `/summarize_hint` is now interactive — invoked without arguments it opens a picker (On / Off / Level 1 / Level 2) like other slash commands. Each level is set individually through an inline input whose label and placeholder state the expected range; after setting one level the picker reopens so the other can be adjusted. Inline arguments still work as a shortcut. Custom-input picker options can now declare their own input label/placeholder (previously hardcoded to /review's "Your instructions").
- Added: docs site now has a full Simplified Chinese version under `/zh/`, with browser-language detection on the homepage and a persistent EN/中文 switch in the top navigation.
- Added: Shells panel — a `Shells (N running)` badge above the input box and a `/shells` command open a picker of tracked background shells (`x` stops one, Enter opens a live detail tab with the log tail and a clickable stop button). Clicking a `bash_background` entry in the transcript jumps to the same tab.
- Changed: synchronous `bash` timeouts no longer kill the command — the process is moved to a tracked background shell and keeps running; the tool returns the partial output plus the shell id to poll with `bash_output`. User interrupts still kill. The timeout parameter is now documented as a synchronous wait budget.
- Changed: stopping a shell or killing a sub-agent from the UI no longer wakes an idle agent. The notice is queued and delivered together with whatever wakes the agent next (your next message included) — natural completions and failures still wake it. These pending notices also no longer block `/summarize` and `/compact`.
- Changed: when `/summarize` or `/compact` is blocked by running background shells, the message now lists the shells and points to `/shells`.
- Added: `/summarize_hint` command — manage the two-tier context summarize reminders: `on`/`off`, or set the trigger levels with `/summarize_hint <level1> <level2>` (integers, 0 < level1 < level2 < 85). Persisted in settings.json (`summarize_hint`) and applied live. New defaults: 50% / 75% (was 60% / 80%), and the reminder prompts now ask for the user's permission before the agent summarizes on its own — autonomous summarization requires consent given in conversation or a standing policy in AGENTS.md.
- Changed: summaries are now ordinary context — any summary (including /summarize-created ones) can be re-summarized and merged like other context groups. A summary belongs to the turn of the nearest preceding user message, so the /summarize picker is a pure turn list (no separate "(Summary)" entries), and adjacent summaries whose covered turns are gone can be merged in one operation.
- Changed: when /summarize covers user messages, their original words are preserved verbatim inside `<user-message>` blocks in the summary (attached file contents are still summarized as data); re-summarization carries these blocks through unchanged. The "exceptionally long user message may be tightened" exception is removed.
- Changed: system notices and peer messages are no longer protected like user messages — the agent may summarize them; only the user's own messages anchor turns and stay untouchable.
- Refactored: session runtime split into 7 sub-modules (`src/session/`) — rewind engine, session log store, context manager, compact prompts, sub-agent factory, child session manager, and session persistence. `session.ts` reduced from ~8100 to ~5865 lines. Session restore no longer constructs a shadow Session; the restore pipeline is now pure-parse (`parseRestoredState`) + single-pass apply, sharing log surgery functions with the live interrupt path. No user-facing behavior change.

## v0.3.7

- Added: system prompt now shows the model identity (e.g. "Anthropic/Claude Sonnet 4") and session start time in the opening paragraph — the model knows who it is before reading any instructions. Model name is snapshotted at session creation and stable across `/model` switches and resumes.
- Added: command picker custom-input options — options marked `customInput` open an inline text field on Enter (like the ask panel's "Enter custom answer"). `/review`'s "Custom review instructions" now uses this instead of showing an error.
- Changed: picker note input (Tab to add instructions) is now opt-in per command (`allowPickerNote`); only `/review` enables it. Other pickers (model selection, `/resume`, etc.) no longer show the Tab hint.
- Changed: unified overlay/panel UI colors — replaced yellow with accent color across command picker, ask panel, prompt select, prompt secret, and OAuth overlays. Yellow is now reserved for interrupted-turn system entries only.
- Fixed: GitHub Copilot model selection broke after GitHub's 2026-06-01 usage-based-billing migration. The `/model` picker now follows your plan's live `/models` catalog — models GitHub no longer offers you are hidden instead of failing with `model_not_available_for_integrator` (HTTP 400) — routing is by model family so Claude Opus 4.7/4.8 and GPT-5.5 no longer throw "Unknown Copilot model", and the gateway API version is bumped to `2026-06-01`.
- Removed: Copilot per-model premium-request "multiplier" labels and the Copilot usage card. Copilot moved to usage-based billing (AI Credits, charged by token), so the legacy premium-request quota no longer applies to most accounts.
- Fixed (security): on case-insensitive filesystems (default macOS, Windows Git Bash) uppercase or wrapper-prefixed dangerous commands (`RM -rf ~`, `ENV rm -rf ~`, `NICE rm -rf ~`) bypassed the danger/catastrophic gate and could auto-run — command-name matching is now case-folded at every layer (wrapper stripping, classification, and the cp/mv escalation). Windows `format`/`diskpart` and PowerShell `Format-Volume`/`Clear-Disk`/`Remove-Partition` now classify as catastrophic (still prompted in yolo mode).
- Fixed: Windows self-update never completed — the staged-update PowerShell helper aborted before copying files because its inner function reused the read-only automatic variable `$Pid` as a parameter, and `Start-Process -ArgumentList @()` is rejected by Windows PowerShell 5.1 on the common no-extra-args relaunch. Renamed the parameter and conditionally omit `-ArgumentList` when empty.
- Fixed: on Windows, shell and `bash`-tool spawns could intermittently lose `git`/`node`/`npm` from `PATH` — the env builder injected a second, truncated `PATH` key alongside the inherited `Path` (case-mismatch) and the truncated one could win. The guard is now case-insensitive.
- Fixed: Windows background processes (`npm run dev`, vite, watchers) leaked as orphans on exit / `/clear` / rewind because `taskkill` was sent without `/F` (a no-op for windowless console processes). It now always force-kills the whole tree.
- Fixed: editing CRLF (Windows-authored) files — `read_file` shows LF-only content, so a multi-line snippet pasted back as `old_str` never matched the CRLF file. `write_file`, `edit_file`, and append now match against and preserve the file's existing line endings, with majority-vote EOL detection so a lone stray CRLF doesn't flip an LF file.
- Fixed: Linux clipboard now cascades wl-clipboard → xclip → xsel → OSC 52 instead of locking to one tool by display server (Wayland sessions with only xclip installed previously failed). OSC 52 wraps for GNU screen, sends raw through tmux so tmux's own `set-clipboard` forwards it, and reports honest failure when stderr isn't a TTY. Windows clipboard image reads (Ctrl+V) no longer freeze the UI (async spawn).
- Fixed: Windows `@`-file completion (backslash/slash normalization), PowerShell non-ASCII output mojibake (forces UTF-8 console encoding for both Windows PowerShell 5.1 and pwsh 7+, while leaving `using`/`param`-leading commands untouched), and hooks invoking `.cmd`/`.bat` shims (npm/npx/prettier) now run through the shell while native `.exe` hooks bypass it. Ctrl+Y added as an image-paste shortcut for Windows Terminal (which consumes Ctrl+V).
- Fixed: external-path permission rules are matched case-insensitively on case-insensitive filesystems, so an approved location isn't re-prompted or hard-failed due to drive-letter/path casing.
- Changed: all three prebuilt native packages (`@opentui/core-*`) are now optional dependencies, so install no longer fails on platforms without a published prebuilt; unsupported platforms get a clear "not supported" message while genuine load failures on supported platforms surface their real error. The `read_file` long-line hint now adapts to the active shell (bash vs PowerShell).

## v0.3.6

- Fixed: Windows startup deadlock — the staged-update PowerShell helper had `$ErrorActionPreference` before `param()` (a parse error), causing the handoff to silently fail and `~/.fermi/staged/` to persist, locking every subsequent launch into a silent `process.exit(0)`. Also renamed `$Args` to avoid shadowing PowerShell's automatic variable.
- Added: handoff marker guard (`.update-handoff-pending`) so a failed PS1 handoff self-heals on the next launch instead of looping forever.
- Added: `/review [prompt]` slash command for code review. Picker offers Uncommitted changes, Against base branch, Specific commit, or Custom instructions. Review prompt uses P0–P3 severity levels and 8-point bug qualification criteria.

Shell detection is fully automatic with priority Git Bash → pwsh → powershell. - Changed: codex (`openai-codex`) requests now mirror the official Codex CLI (`codex_cli_rs`) wire contract instead of an experimental opencode spoof. The request carries an honest `originator: fermi`, a `fermi/<version> (<os> <release>; <arch>)` `User-Agent` (version lives only in the UA — the CLI sends no separate `version` header), the real `ChatGPT-Account-Id` extracted from the OAuth access token's JWT, and `session_id` carrying the prompt-cache key. The non-standard `conversation_id` header (which the CLI never sends) was dropped. Verifying against the open-source Codex CLI showed the backend accepts arbitrary `originator` values (`codex_cli_rs`, `opencode`, `openclaw` all work), so the spoof had no benefit.
- Changed: codex requests now strip fields the Codex backend silently degrades on rather than rejecting — `temperature`, `max_output_tokens`, `top_p`, `metadata`, `prompt_cache_retention` — none of which the official Codex CLI sends. These were already skipped on the default path, but a user-supplied `extra` block could reintroduce them; the strip now runs last so nothing leaks through. Fields the CLI does send (`prompt_cache_key`, `service_tier`, `text`, `reasoning`, `include`, `store`) are preserved.
- Fixed: encrypted reasoning was dropped on the OpenAI Codex backend, so the chain of thought was never carried across tool-call rounds. The streaming parser only reconstructed reasoning from `response.completed.output`, but the codex backend (`chatgpt.com/backend-api/codex`) returns an empty `completed.output` and delivers the encrypted reasoning item only through `response.output_item.added/done` events. Reasoning capture now reads those streamed output items (matching the Vercel AI SDK's approach) for both `openai` and `openai-codex`, so `reasoning.encrypted_content` is preserved and replayed. The reasoning round-trip payload is now reasoning-only — function calls are replayed via the existing `tool_calls` path instead of being bundled into the sealed payload (which previously double-sent them); old logs that stored `[reasoning, function_call]` payloads still replay correctly.
- Fixed: the OpenAI (`provider: "openai"`) Responses path lost its reasoning between tool-call rounds and missed reasoning-related cache hits. Fermi manages the conversation client-side and never sends `previous_response_id`, but the openai provider was left on the default `store: true` *without* requesting `include: ["reasoning.encrypted_content"]` — so server-side reasoning was never read back (no `previous_response_id`) and no encrypted reasoning was echoed either, leaving only the lossy `summary`, which the backend discards. The openai provider now goes fully client-managed stateless like `openai-codex` and `copilot`: it forces `store: false`, requests encrypted reasoning, and replays the sanitized encrypted `reasoning` round-trip items each turn. This preserves the chain of thought across tool rounds and restores reasoning-token cache reuse. `openai` and `openai-codex` now take the exact same reasoning path (OpenRouter stays separate because of its Fernet re-encryption envelope).
- Providers / model picker: added native Qwen / DashScope support on the OpenAI-compatible Responses path with three managed regional endpoints (`qwen` China, `qwen-intl` Singapore, `qwen-us` Virginia), plus OpenRouter presets for `qwen/qwen3.6-plus` and `qwen/qwen3.7-max`. Qwen direct defaults now use DashScope's documented Responses base URLs under `/api/v2/apps/protocols/compatible-mode/v1`; request shaping uses DashScope Responses controls (`enable_thinking` and built-in `{ type: "web_search" }`), replays prior plaintext thinking as Responses `reasoning` items, captures streamed `web_search_call` sources as citations, routes detected `DASHSCOPE_*` / `QWEN_*` keys through the managed import flow before treating the provider as configured, capability tables know the two models (1M context, thinking, Qwen3.6 Plus multimodal), and TUI model labels/vendor buckets render them with beautified `Qwen` names instead of raw ids.
- Fixed: queuing a follow-up message while the assistant was mid-turn could later crash a long session with a provider 400 (`messages[N].content: invalid type: sequence, expected a string` on DeepSeek and other Anthropic-protocol backends). Draining the queued message advanced the turn counter while a provider round was still in flight, so that round's streamed reasoning was stamped with a different turn index than its own tool calls; the projection then grouped the reasoning into a standalone, content-less "thinking-only" assistant message that strict backends reject. Each activation now stamps all of its log entries (reasoning, text, tool calls, etc.) with a single turn index captured at the start, so a mid-flight drain can no longer split a round. Sessions already broken by this are self-healed on load: an orphaned reasoning entry whose round's real entries sit under one other turn is re-stamped to match.
- Added: tool-input repair for open models. Recoverable argument-shape mistakes that DeepSeek / GLM / Qwen / Kimi routinely make are now fixed at the exact failing argument instead of being rejected — a stringified array (`'["a","b"]'`), an object placeholder, or a bare string where a list was expected are coerced to the right shape; a markdown auto-link leaked into a file path (`"[notes.md](http://notes.md)"`) is unwrapped to the plain path. Repairs only run after the as-is value fails validation, so valid inputs (including `write_file` content that merely looks like JSON or markdown) are never touched.
