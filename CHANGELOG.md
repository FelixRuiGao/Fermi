# Changelog

All notable changes to Fermi are recorded here. Releases that pre-date
this file are on the [GitHub Releases page](https://github.com/FelixRuiGao/Fermi/releases).

The release workflow extracts the section whose heading exactly matches
the pushed tag (e.g. `## v0.3.2-alpha.3`) and uses it as the GitHub
Release notes. A missing or empty section fails CI.

## Unreleased

- Changed: codex (`openai-codex`) requests now carry the three client-identification headers every mature codex client sends — `ChatGPT-Account-Id` (extracted from the OAuth access token's JWT so the request bills/routes through the user's actual ChatGPT account), `originator`, and `User-Agent`. Without these the codex backend treats the request as an unknown third-party client and appears to apply heavier throttling. `originator` and `User-Agent` currently spoof opencode (`opencode` / `opencode/1.15.10 (...)`) on the hypothesis that the backend gives well-known clients better treatment; this is an experiment and is easy to flip back to a `fermi` identity if it doesn't help.
- Fixed: encrypted reasoning was dropped on the OpenAI Codex backend, so the chain of thought was never carried across tool-call rounds. The streaming parser only reconstructed reasoning from `response.completed.output`, but the codex backend (`chatgpt.com/backend-api/codex`) returns an empty `completed.output` and delivers the encrypted reasoning item only through `response.output_item.added/done` events. Reasoning capture now reads those streamed output items (matching the Vercel AI SDK's approach) for both `openai` and `openai-codex`, so `reasoning.encrypted_content` is preserved and replayed. The reasoning round-trip payload is now reasoning-only — function calls are replayed via the existing `tool_calls` path instead of being bundled into the sealed payload (which previously double-sent them); old logs that stored `[reasoning, function_call]` payloads still replay correctly.
- Fixed: the OpenAI (`provider: "openai"`) Responses path lost its reasoning between tool-call rounds and missed reasoning-related cache hits. Fermi manages the conversation client-side and never sends `previous_response_id`, but the openai provider was left on the default `store: true` *without* requesting `include: ["reasoning.encrypted_content"]` — so server-side reasoning was never read back (no `previous_response_id`) and no encrypted reasoning was echoed either, leaving only the lossy `summary`, which the backend discards. The openai provider now goes fully client-managed stateless like `openai-codex` and `copilot`: it forces `store: false`, requests encrypted reasoning, and replays the sanitized encrypted `reasoning` round-trip items each turn. This preserves the chain of thought across tool rounds and restores reasoning-token cache reuse. `openai` and `openai-codex` now take the exact same reasoning path (OpenRouter stays separate because of its Fernet re-encryption envelope).
- Providers / model picker: added native Qwen / DashScope support on the OpenAI-compatible Responses path with three managed regional endpoints (`qwen` China, `qwen-intl` Singapore, `qwen-us` Virginia), plus OpenRouter presets for `qwen/qwen3.6-plus` and `qwen/qwen3.7-max`. Qwen direct defaults now use DashScope's documented Responses base URLs under `/api/v2/apps/protocols/compatible-mode/v1`; request shaping uses DashScope Responses controls (`enable_thinking` and built-in `{ type: "web_search" }`), replays prior plaintext thinking as Responses `reasoning` items, captures streamed `web_search_call` sources as citations, routes detected `DASHSCOPE_*` / `QWEN_*` keys through the managed import flow before treating the provider as configured, capability tables know the two models (1M context, thinking, Qwen3.6 Plus multimodal), and TUI model labels/vendor buckets render them with beautified `Qwen` names instead of raw ids.

## v0.3.3

- TUI: added `/diff` to choose write/edit diff display. Compact remains the default; Full persists globally and expands file-modify blocks inline instead of showing the earlier-lines fold prompt.
- TUI: markdown H1/H2 heading colors changed from orange/gold to blue tones (closer to the accent palette).
- Fixed: model selection was not remembered across restarts — switching with `/model` always reverted to the model picked during first-time setup. The init wizard wrote that initial pick into `settings.json` as `default_model`, which is a *declarative pin* that overrides `state/model-selection.json` (the system-managed last-used memory) on every startup; `/model` only updates the latter, so the pin always won. The wizard no longer writes `default_model` — the initial selection is still persisted to `model-selection.json`, and `default_model` is now purely opt-in for users who want to hard-pin a model by hand. Existing `settings.json` files that already contain an auto-written `default_model` should remove that line for model memory to take effect.
- Fixed: Kimi (Moonshot) on the Anthropic transport crashed every web-search turn with `JSON Parse error: Unexpected EOF`. On a degenerate (empty) search — which Kimi K2.6 prepends to even trivial turns like a bare greeting — Kimi emits an `input_json_delta` SSE event with **no** `partial_json` field, and the Anthropic SDK's stream accumulator computes `"" + undefined === "undefined"` and feeds that to its partial-JSON parser, which throws and kills the whole turn. Kimi's client now wraps fetch with a small SSE repair (`makeAnthropicSSERepairFetch`, driven by `TransformStream`/`pipeThrough`) that normalizes any `partial_json`-less `input_json_delta` to an empty string before the SDK sees it; real searches (which carry a proper `partial_json`) and every compliant vendor pass through untouched. Real Kimi web search now completes end to end and returns live results.
- Fixed: Kimi's leading "Search results for query: ..." preamble was suppressed from the *saved* transcript but still streamed to the live TUI on every search turn. `_callStream` now buffers a leading text block and drops it if a `server_tool_use` web search immediately follows (Kimi-only, gated on web-search support), so the preamble never reaches the live stream; the real answer — which always follows thinking/search — still streams incrementally.
- Prompt caching: on the Anthropic provider, `cache_control` breakpoints now cover the tool definitions and the last message in addition to the system block (previously only the system block), so tool schemas and the conversation prefix are cached too, not just the system text. Session-specific paths (`{PROJECT_ROOT}` / `{SESSION_ARTIFACTS}` / `{SYSTEM_DATA}`) are substituted inline with their real absolute values, so tool-call examples and instructions always show concrete paths the agent can use directly; the substituted prompt is stable within a session (and across sessions in the same project), so it stays cache-friendly turn-to-turn.
- Prompt: a short `# Session Configuration` section is appended at the very end of the system prompt, carrying the session start timestamp (rendered in the runtime's local timezone). It is labelled as the time of the first message — explicitly the *start* time, not necessarily "now", since a session can be resumed after an arbitrary gap — and points the agent at the `time` tool for the current time. It is provided as ambient context only; the agent is not instructed to comment on it. Spawned sub-agents each append exactly one such section with their own start time — the parent does not pre-render it into the child's template prompt.
- Agent prompts: reworked the main agent prompt. Dropped the separate communication/tone section; kept the four-phase workflow (Explore → Plan → Act → Review, iterative) with guidance to delegate bulk exploration to `explorer` sub-agents and protect their independent context, and added a note to reach for external sources (web search, official docs, discussions) when stuck instead of retrying in a vacuum. A "Your judgment" section covers contributing views, pushing back before committing to the user's call, and building only what was agreed; a "Mindset" note covers staying honest under pressure and never fabricating success. Sub-agent delegation guidance distinguishes orienting context (goal, constraints, facts) from contaminating a fresh sub-agent with your own conclusions and dead ends, and reviewers read `git diff` as the unbiased source of truth instead of a prose summary.- TUI: quitting fermi no longer leaves SGR mouse-event garbage (e.g. `51;790;1276M`) on the shell prompt when mouse motion was occurring during exit. The fix has three layers: (1) destroy now writes `?1000l/?1002l/?1003l/?1006l` to actually stop mouse reporting — upstream OpenTUI 0.2.1 omitted this and got away with it because modern terminals like iTerm2/Alacritty/Kitty/Ghostty implicitly clear mouse modes when the alt-screen client exits, but macOS Terminal.app does not; (2) fermi's own SGR-Pixels feature added `?1016h` for sub-cell scrollbar drag but never paired it with `?1016l` on shutdown, so SGR-Pixel framing stayed armed (zig's setMouseMode deliberately ignores `?1016`); and (3) even after writing every reset CSI, one or two bytes still leaked because mouse-motion reports already sitting in the kernel TTY input buffer were inherited by the shell when raw mode was restored. `main.tsx` now resets mouse tracking, waits ~30 ms so the kernel pumps in-flight bytes up into node where the still-attached stdin parser consumes them, drains node's readable buffer, and only then destroys the renderer. All reset CSIs go directly through the TTY rather than zig's `writeOutBuf` to avoid being dropped when the render thread suspends.
- TUI: prompt history navigation now only triggers at absolute composer boundaries. After recalling a history item, ↑/↓ first move through the recalled text normally; history only advances again from the start/end of the composer, while the existing live-draft restore semantics are preserved.
- TUI: OpenAI Codex OAuth token refresh is now active-model lazy instead of scanning every configured `openai-codex` entry during startup. If refresh fails, the error is written into the conversation as a normal session error entry instead of leaking through stderr into the TUI input area. Switching or logging into Codex also invalidates cached model configs so the active provider picks up the fresh access token.
- Model switching: `/model` now switches the active model in place instead of resetting into a fresh conversation. Thinking history is tracked on three independent axes — transport protocol (`responses` / `anthropic` / `chat`), encryption family (`openai` / `anthropic` / `none`), and sealed wire schema (`anthropic-messages` / `openai-responses` / `openrouter-chat`). Sealed reasoning round-trips only when **both** the encryption family and the schema match: Anthropic direct and Copilot Anthropic share `anthropic-messages` (verified empirically), OpenAI direct / Copilot Responses / openai-codex share `openai-responses`, but OpenRouter wraps reasoning in its own Fernet-encrypted envelope (`openrouter-chat`) that is not interchangeable with direct OpenAI even when both target the OpenAI family. Cross-family or cross-schema sealed reasoning is omitted; non-encrypted targets receive a plain-text replay of the prior thinking summary. Legacy OpenRouter sessions saved before this change now have their `reasoning_details` recognized on first replay via shape inference (encryption hint inferred from `signature` / `reasoning.encrypted` entries).
- Providers: restored Kimi/Moonshot native Anthropic web search (`web_search_20250305`) on the new Anthropic transport path, unified provider default base URL fallbacks in a shared table (including `kimi-code`), tightened Anthropic Messages request shaping so Chat/Responses-only fields like `extra_body`, `reasoning_effort`, `web_search_options`, `top_p`, and `top_k` no longer leak into Anthropic requests, and filtered the leading "Search results for query: ..." preamble text block that Kimi emits before `server_tool_use` (the preamble is now dropped from the agent's text field so the saved transcript only contains the real answer).
- Providers: migrated Kimi/Moonshot, MiniMax, DeepSeek, and Xiaomi MiMo from OpenAI-chat-compatible adapters to Anthropic Messages-compatible adapters, with new `/anthropic` default base URLs, shared Anthropic protocol handling, vendor-specific thinking controls, and deprecated rollback stubs for the old chat adapters.
- Tools: improved client-side `web_search` / `web_fetch`. Exa MCP search fallback now parses structured text into individual titled URL results with multiline `Highlights:` blocks instead of returning one aggregated empty-URL result or collapsing evidence to the first line; long highlights are soft-truncated at nearby word boundaries. `web_fetch` still uses Jina Reader first, but local fallback now rejects local/private targets, limits redirects, converts HTML through Readability + Turndown, and normalizes successful output to `# Content from <url>` plus readable page content.
- TUI: assistant markdown now renders with top-level block boundaries while reusing same-kind renderables during streaming. This keeps the top-level one-gap spacing model for headings, paragraphs, tables, code blocks, and blockquotes, avoids stale height floors when an incomplete table paragraph becomes a real table, and prevents the old top-level flicker caused by destroying/recreating ordinary text blocks on every streamed update.
- Auto-update: unified all platforms on a staged-update flow. `fermi update` now downloads and stages the latest release for the next restart instead of replacing the live binary inline. Release asset naming and executable names are now sourced from the platform binary-asset metadata rather than updater-local string literals. On Windows, staged updates are handed off to a bundled PowerShell helper that waits for `fermi.exe` to exit before replacing files and relaunching, avoiding self-overwrite of the running executable. Public docs and install/update docs now describe macOS (Apple Silicon), Linux (x86_64), and Windows (x64) consistently.
- `/resume` / `/session`: restoring a session now forces a full screen repaint. A terminal left visually desynced (e.g. a garbled markdown table or a heading "merged" into the table above, after scrolling during streaming) is now cleaned up on resume — previously only a terminal resize recovered it.
- Built-in skills: added 5 bundled document & data skills — `pdf`, `docx`, `xlsx`, `pptx`, and `csv-data`. Invoke any with `/<name>` or let the model pick by description; each installs its Python libraries (pypdf / python-docx / openpyxl / python-pptx / pandas) on demand and degrades gracefully with an install hint if unavailable.

## v0.3.2

- Init wizard: Ctrl+C on the first step now exits cleanly instead of printing a stack trace.
- Init wizard: selecting a model no longer loops back to the picker with a "Done" confirmation — selection proceeds directly to the next step (thinking level). Ctrl+C still goes back.
- Init wizard: sub-agent tier model picker now shows all providers (matching the main model picker), not just already-configured ones. Selecting an unconfigured provider triggers inline credential setup.
- Auto-update overhaul: structured `UpdateState` replaces the old string-based notice system. TUI now shows a hint in the status bar when an update is downloaded (`✓ v0.3.2 ready (restart to apply)`) or available (`v0.3.2 available — run fermi update`). Version comparison correctly handles prerelease suffixes. Tarball downloads are verified against a `.sha256` checksum when available. Binary replacement uses atomic copy→rename. Dev mode is detected and skips auto-apply to prevent writing to wrong directories. Disk version is checked before applying staged updates to prevent duplicate writes from concurrent instances.
- Auto-update: `auto_update` setting now accepts `true` (default), `"notify"` (check but never download), or `false` (disable). With `true`, patch and minor updates auto-download and stage; major updates only notify. This is a behavior change from the previous `true` which auto-downloaded all versions.
- `fermi update --check`: check for updates without downloading or installing. Works in dev mode too (only the install path requires a production install).
- TUI: redesigned welcome screen — the old ASCII block logo (top-left, multi-hue vertical gradient) is replaced by a centered "Fermi Here." text identity with a smooth color-breathing animation (pale blue ↔ blue-purple, compound sine ~4s period). Below it: version, cwd, and key hints (`/` commands · `@` files · `/help` shortcuts) — only information not already shown by the input status bar. The welcome stage is absolutely positioned against terminal height, so it stays perfectly still when slash/command panels open (the old logo jumped because it was centered inside the shrinking scrollbox viewport). ANSI Shadow figlet art kept in theme tokens for potential future use; sidebar wordmark + system entries continue to use their own gradient (decoupled).
- TUI: the scrollbar thumb is now the correct length immediately after `/resume` (or any restore into a previously empty transcript), instead of collapsing to the ~1-cell minimum until the next window resize or slash-command panel toggle. Root cause: on the empty TUI's first layout, `viewport.onSizeChange` fired before `content.onSizeChange`, so `Slider.viewPortSize` was set while `slider.max` was still 0 and got clamped to `0.01` by the `min(size, max - min)` guard; the subsequent content growth updated `max` but the unchanged `ScrollBar.viewportSize` short-circuited its setter, so the stale `0.01` was never re-synced. `slider.viewPortSize` is now assigned inside `updateSliderFromScrollState()` after `slider.max`, so the clamp always sees a current range.
- TUI: markdown after a table (or other separately-rendered block) that begins with a `---` thematic break no longer renders as raw, yellow-tinted text. The remaining markdown was coalesced into a synthetic sub-block starting with `---`, which tree-sitter parsed as YAML front matter — so every heading/list/bold between that `---` and the next was shown as un-concealed directive-coloured text instead of formatted markdown. The synthetic block is now re-highlighted with a leading newline (offsets shifted back) so a leading `---` is parsed as a thematic break, not front matter. Covers the monkey-patched render path the app actually uses, with and without blank-line separators.
- TUI: streaming markdown no longer makes the whole transcript jump up a line and back when an inline span closes (e.g. `**bold**`). The reflow from raw markers to formatted text is horizontal as intended; previously a one-frame height dip in the streaming block — before async highlight/conceal settled — was amplified by sticky-bottom auto-follow into a vertical bounce of the entire transcript. The streaming block now holds a per-wrap-width height floor while streaming (monotonic, reset when the message finalizes), so a transient measurement can no longer shrink it mid-stream.
- TUI: user messages get a faint accent background wash plus a solid 1-column accent bar flush against it, replacing the bar-only marker that was hard to spot while scrolling. The wash is a low-alpha accent (`userWash` palette token, dark and light) — distinct enough to anchor the eye when scanning back through a long transcript, without the weight of the old solid surface. Queued (pending) messages use the same shape, kept distinct by dim text.
- TUI: scrollbar thumb now renders at 1/8-cell precision (eighth-block characters) instead of 1/2-cell, so the thumb glides smoothly along the rail line-by-line instead of jumping in half-cell increments on long conversations. Added a dedicated `scrollbarThumb` palette token alongside the existing `scrollbarTrack` so the rail (now a solid mid-grey instead of empty space) and the thumb (matches body-text colour) are visually distinct. Default minimum thumb size drops from 4 cells to 1 cell, matching the look of other agent terminals like amp; on very long content the thumb is small but stays buttery-smooth as it moves. Dragging the thumb with the mouse is now sub-cell smooth too on terminals that support SGR-Pixels mouse reporting (kitty, Ghostty, WezTerm, iTerm2, recent xterm, VS Code): ScrollBar no longer rounds the scroll position to integer cells, and when the terminal reports the mouse in pixels the position is decoded at fractional-cell resolution. Terminals without pixel mouse reporting (e.g. macOS Terminal.app) fall back to the previous cell-quantized drag with no change in behaviour.
- TUI: switching to a detail tab (clicking a thinking/tool entry) and back no longer resets the transcript to the bottom or clears the composer. The main view now stays mounted with Yoga `Display.None` instead of being unmounted, so scroll position and textarea contents survive tab switches. As part of the same change the composer is now anchored to the bottom of the screen at all times — previously it lived inside the scrollbox and would scroll off-screen when reading history; now scrolling history keeps the input visible, and typing no longer auto-scrolls the transcript to the bottom.
- TUI perf: cut shiki preload from 41 languages to 9 (typescript/tsx/js/jsx, python, bash, json, markdown, diff); everything else lazy-loads on first encounter, briefly flashing a highlight.js fallback while loading. Combined with a synchronous `Bun.gc()` at turn boundaries (throttled to once per 10 s on completed turns), startup peak memory drops by ~130 MB and one observed long session reclaimed ~160 MB the instant a turn ended.
- TUI: body text now follows the terminal's configured foreground (OSC 10) in auto theme mode — so dark/light terminal themes give matching text colour without per-theme tuning. Pinned `FERMI_THEME=light|dark`, or OSC-unresponsive terminals, keep the token-table palette as a contrast-safety fallback.
- TUI: tool results (`list`, `glob`, `grep`, `bash`, `bash_output`, `web_search`, `web_fetch`, `send`, `apply_edits`) now collapse to a single truncated line with a clickable "N more lines, CLICK to open" fold indicator. Errors still expand to 8 lines. Reduces visual noise during long sessions.
- TUI: input prompt `❯` and textarea cursor changed from accent blue to neutral white — works better across terminal palettes and stops fighting with the brand accent.
- Permissions: compound commands with a trailing redirect (e.g. `cd dir && npm install 2>&1`, `pnpm install && pnpm build > build.log`) now offer "Always allow" for the inner command just like the same command without the redirect. Previously the redirect forced a one-time approval.
- GUI: replaced the three separate Models/Skills/Integrations modals with a single unified Settings dialog (left-nav rail + scrollable right content). Six sections: General, Models, Providers, Skills, MCP Servers, Hooks. Opens via the sidebar footer menu or the new `⌘,` shortcut.
- GUI: real CRUD in Settings now writes through to `~/.fermi/settings.json` — Add/Edit/Delete MCP servers via inline form (name / command / args / env / URL); change Default model / Thinking level / Permission mode from dropdowns. Skills toggles use a right-aligned iOS-style switch.
- GUI transcript: `write_file` / `edit_file` tool calls now render as a card with file-type icon, +N/−M diff stats, and an inline unified diff with line numbers and red/green rows (replaces the prior `path +N` chip).
- GUI transcript: code blocks gained a hover "Copy" button. Reasoning blocks ("Thinking") gained a soft left-accent stripe.
- GUI composer: model picker is grouped by provider with brand labels and capability icons (thinking, multimodal); active model badged. Permission picker shows a colored status dot (safe / default / risky). Token counter tints to warning then error as context fills. `@path` reference hint surfaced in the placeholder. `Esc` interrupts a running turn.
- GUI sidebar: top-level "New session ⌘N" button + `⌘N` keybinding; hover pin overlaps fixed; "Show all N" affordance got a chevron.
- GUI session backend: new RPCs `session.getSummarizeTargets` / `session.getContextIdsForTurnRange` so the GUI's Summarize dialog can drive the same picker flow the TUI's `/summarize` command uses.
- GUI session backend: new IPC `settings.upsertMcpServer` / `settings.deleteMcpServer` / `settings.updateDefaults` to support the inline Settings forms above.
- GUI transcript: expanded tool-call output (bash / read_file / etc.) gained a hover "Copy output" button, matching the markdown code-block pattern.
- GUI right-pane empty states: Plan, Agents, and Shells panels each gained a one-line explainer ("checkpoints come from plan.md", "sub-agents appear here when spawned", "bash_background processes appear here") so empty panels are self-documenting.
- GUI right-pane Files: filter input got an X clear button — matches the Settings search inputs.
